## 📌 회의 내용

### 📝 사용자 요구사항 정의서
화면 정의서를 작성하려고 보니 사용자 요구사항 정의서 수정할 것이 보여 최종적으로 수정했다.

### ✏️ 깃 푸쉬 방식

이전에는 각자 팀장의 레포지토리를 포크해서 작업 후 PR을 날리는 방식이었지만, 이 경우 원격 레포지토리를 계속해서 `sync fork` 해줘야 하기 때문에 방식을 바꾸었다.

> 각 팀원은 팀장 레포지토리를 클론받는다.
깃 브랜치를 생성하고 작업 후 팀장 레포지토리에 push한다.
dev 브랜치에 PR을 날리고 토의 후 병합한다.


### 🖥️ 개발 분담
먼저 모든 페이지를 분할해서 개발하는 것이 아니라, 공통 컴포넌트를 각자 하나씩 맡아 월요일까지 개발 후 다시 이야기 해보기로 했다.
