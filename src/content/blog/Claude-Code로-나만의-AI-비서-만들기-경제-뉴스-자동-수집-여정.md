> "클로드 코드를 활용해서 AI 비서같은 걸 만들 수 있을까?"
> 그 질문에서 시작된 삽질과 발견의 기록입니다.

## 🤔 시작: 개발자도 경제 공부가 필요하다

요즘 들어 경제 뉴스에 관심이 생겼습니다. 주식이든 부동산이든, 개발자도 경제 흐름은 알아야 하지 않을까 싶어서요.

그런데 매일 아침 뉴스 사이트 돌아다니며 기사 읽고, 모르는 용어 검색하고... 솔직히 번거롭습니다.

> "자고 있는 동안 AI가 뉴스 정리해서 보내주면 안 될까?"

Claude Code를 쓰면서 자연스럽게 든 생각이었습니다.

## 🎯 목표 정의

단순 뉴스 요약이 아니라, **경제 공부**가 목표였습니다:

- 매일 아침 주요 경제 뉴스 5개 수집
- 각 뉴스에 대한 심층 분석 (시장 영향, 수혜/피해 산업)
- 모르는 경제 용어 자동 설명
- Slack으로 자동 전송

"크론잡으로 매일 아침 자동 실행되면 좋겠다"가 최종 그림이었습니다.

## 🔄 4가지 접근법 비교

Claude Code로 자동화하는 방법을 찾아보니, 크게 4가지 방식이 있었습니다:

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **1. 크론잡 + CLI** | `claude -p "..."` 쉘 스크립트 | 간단, 기존 구독 활용 | 웹 검색 제한적 |
| **2. Anthropic API** | 직접 API 호출 | 유연함 | **비용 발생** |
| **3. MCP 서버** | 커스텀 MCP 구축 | 확장성 | 복잡함 |
| **4. GitHub Actions** | 스케줄 워크플로우 | 무료, 관리 편함 | 환경 제약 |

> "API 직접 호출하면 비용이 발생하겠네?"

맞습니다. 2번 방식은 매번 토큰 비용이 들어갑니다. 이미 Claude Code 구독하고 있으니, **1번 방식(크론잡 + CLI)** 으로 시작하기로 했습니다.

나중에 더 복잡한 기능이 필요하면 그때 MCP로 확장하면 됩니다.

## 🐳 1차 시도: WebFetch로 뉴스 가져오기

"매일경제 사이트에서 뉴스 가져오면 되겠지?"

```bash
# 첫 번째 시도
claude -p "mk.co.kr/news/economy 에서 오늘 뉴스 5개 요약해줘"
```

```
❌ Claude Code is unable to fetch from www.mk.co.kr
```

음, WebFetch가 막혔습니다. 사이트 측에서 봇 접근을 제한하는 것 같습니다.

## ☸️ 2차 시도: 웹 검색으로 우회

WebFetch가 안 되니 웹 검색을 해보기로 했습니다.

```bash
claude -p "웹 검색해서 오늘 경제 뉴스 찾아줘"
```

검색은 되는데... 문제가 있었습니다.

```
🤯 오늘은 2026년 2월 2일인데 옛날 자료를 찾네
```

웹 검색 결과가 최신 뉴스가 아니라 과거 기사들이 섞여 나왔습니다. 날짜 필터링이 제대로 안 되는 것 같습니다.

## 🤯 현타: 뭘 하고 있는 거지?

```
✅ WebFetch 시도 → 실패
✅ 웹 검색 시도 → 날짜 문제
🔄 다른 방법 찾는 중...
❌ 아직 뉴스 한 줄도 못 가져옴
```

**원래 목표**: 경제 뉴스 자동 수집
**현재 상황**: 뉴스 접근 자체가 안 됨

## 🔄 방향 전환: Chrome MCP

> "chrome mcp 사용해서 해봐"

그렇습니다. Claude Code에는 Chrome 브라우저를 직접 제어하는 MCP가 있습니다. 사람처럼 브라우저로 접속하면 되지 않을까?

### Chrome MCP 설치 삽질

```bash
npm install -g @anthropic-ai/mcp-chrome
```

```
❌ npm error 404 Not Found
```

패키지 이름이 다른 것 같습니다. 검색해보니 `mcp-chrome-bridge`였습니다.

```bash
npm install -g mcp-chrome-bridge
```

```
✅ added 207 packages
```

설치 완료! Claude Code 재시작 후...

## 💻 Chrome MCP로 뉴스 수집

이제 진짜 브라우저로 접속합니다:

```markdown
1. tabs_context_mcp로 브라우저 상태 확인
2. navigate로 mk.co.kr/news/economy 이동
3. read_page로 헤드라인 추출
4. 각 기사 클릭해서 본문 읽기
5. 요약 및 분석 생성
```

드디어 작동합니다!

## 🎯 최종 구조: Claude Code 스킬로 만들기

단순 명령어보다 **스킬**로 만들면 재사용이 편합니다.

```
~/.claude/commands/news-summary.md   # Claude 스킬 (프롬프트)
~/scripts/news-summary.sh            # 쉘 스크립트 (스킬 호출)
~/.env.news                          # Slack 웹훅 URL
```

### 스킬 프롬프트 구조

```markdown
## 매일경제 뉴스 요약

### 1단계: 뉴스 수집
- Chrome MCP로 mk.co.kr 접속
- 메인 헤드라인 5개 추출

### 2단계: 심층 분석
각 뉴스에 대해:
- 배경: 관련 정책/사건 맥락
- 시장 영향: 주식/채권/환율 영향
- 수혜/피해 산업: 구체적 업종
- 투자자 관점: 개인 투자자 행동 가이드
- 향후 전망: 단기/중기 시나리오

### 3단계: 경제 용어 정리
초보자도 이해할 수 있게 어려운 용어 설명

### 4단계: Slack 전송
curl로 웹훅 호출
```

### 쉘 스크립트

```bash
#!/bin/bash
source ~/.env.news
claude -p "/news-summary" --allowedTools "Bash,mcp__*"
```

### 크론잡 설정

```bash
# 매일 아침 7시에 실행
0 7 * * * /Users/crowwan/scripts/news-summary.sh
```

## 💡 배운 것들

### 1. "간단하게 시작하자"
처음부터 완벽한 시스템을 만들 필요 없습니다. CLI + 크론잡으로 시작해서, 필요하면 나중에 MCP로 확장하면 됩니다.

### 2. "장애물은 다른 길로"
WebFetch가 안 되면 Chrome MCP, 웹 검색이 안 되면 직접 접속. 방법은 항상 있습니다.

### 3. "비용도 엔지니어링의 일부"
API 직접 호출 vs 기존 구독 활용. 개발 편의성뿐 아니라 운영 비용도 고려해야 합니다.

### 4. "목표를 구체화하면 길이 보인다"
"뉴스 요약"이 아니라 "경제 공부"가 목표라는 걸 명확히 하니, 경제 용어 설명 기능까지 추가하게 됐습니다.

## 마무리

> "클로드 코드를 활용해서 AI 비서같은 걸 만들 수 있을까?"

네, 만들 수 있습니다. 그리고 생각보다 어렵지 않습니다.

**핵심은 완벽한 시스템이 아니라, 작동하는 시스템을 먼저 만드는 것입니다.**

WebFetch 실패, 웹 검색 날짜 이슈, 패키지 이름 오류... 삽질은 있었지만, 결국 원하는 결과물을 얻었습니다.

여러분도 반복적인 작업이 있다면, Claude Code로 자동화해보세요. 생각보다 재미있습니다.
