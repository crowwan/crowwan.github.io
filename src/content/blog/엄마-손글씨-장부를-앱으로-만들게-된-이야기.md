## 🤔 매년 반복되는 연례행사

엄마가 동아리 회계 담당을 맡고 계십니다. 매년 연말이 되면 해야 하는 일이 있어요.

- 엄마는 손글씨로 회계 장부를 정리하시고
- 저는 그걸 한글(HWP) 파일로 만들어드립니다
- 작년 양식을 꺼내서, 올해 데이터 채우고
- 합계 맞는지 확인하고, 표 서식 맞추고...

**매년 같은 작업의 반복입니다.**

올해도 어김없이 그 시간이 왔습니다. 손글씨 장부 사진을 받아서 Claude에게 던졌어요.

> "이거 표로 정리해줘"

---

## 🐳 삽질의 시작

### Excel 시도

Claude가 깔끔하게 Excel로 만들어줬습니다. 나쁘지 않았어요.

근데 뭔가 좀 더 **예쁜 형태**로 만들고 싶었습니다. 이전에 만들던 그 손글씨 장부 느낌.

### 한글 MCP 시도

이전 양식을 주고 한글 파일로 만들려고 했습니다. 예전에 제가 만들어둔 한글 MCP가 있거든요.

그런데... **생각보다 잘 동작을 안 했습니다.** 솔직히 MCP를 그렇게 실용적이지 못하게 만든 것 같아요. ~~(내 탓)~~

그래서 포기.

### HTML로 전환

> "다른 방법으로 만들어줘"

Claude가 HTML로 만들어줬습니다. 브라우저에서 열어보니 **나쁘지 않아요.** 손글씨 장부랑 거의 비슷한 느낌.

이걸로 가기로 했습니다.

---

## ⚠️ 끝없는 수정의 늪

HTML로 형태는 잡혔는데, 문제는 그 다음이었습니다.

| # | 요청 |
|---|------|
| 1 | "항목명 틀렸어. '인원추가제비'가 아니라 '원단구입식대'" |
| 2 | "3월 금액이 8,100원인데 78,100원으로 잘못 됐어" |
| 3 | "월별로 테두리 구분이 필요해" |
| 4 | "좌우 행 개수가 안 맞아" |
| 5 | "합계 계산해봤는데 안 맞는데?" |
| ... | (수십 번 반복) |

항목 하나 바꾸고, 숫자 하나 고치고, 레이아웃 조금 수정하고...

이걸 **수십 번** 반복했습니다.

---

## 🤯 깨달음의 순간

어느 순간 제가 말했어요.

> **"좋아 너도 느끼겠지만, 이거 너무 귀찮잖아."**

생각해보니까요:

| 문제 | 설명 |
|------|------|
| 매년 반복 | 데이터만 바뀌고 형식은 똑같음 |
| 수작업 변환 | 엄마 → 손글씨 → 나 → 한글 파일 |
| 수정 지옥 | 하나 틀리면 다시 요청, 다시 대기 |

> **"엄마가 직접 입력하고 바로 결과물을 볼 수 있으면 어떨까?"**

---

## 🔧 12분만에 앱 만들기

> "이거 앱으로 만들어줘. 계획서부터 작성해줘."

Claude가 몇 가지만 물어봤습니다:

- 실제 배포가 필요한가요? → **필요함**
- 오프라인 환경에서 돌아가야 하나요? → **아니요**

그게 끝이었어요.

**12분 후**, 기획서(PLANS.md)와 함께 동작하는 앱이 나왔습니다.

---

## 📋 완성된 기획서

Claude가 만든 PLANS.md에는 다 담겨 있었습니다:

```
기술 스택: React + TypeScript + Vite + Tailwind CSS
저장: localStorage (서버 없음)
이미지 출력: html2canvas
UI: Mobile First, 탭 네비게이션
```

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 기본정보 입력 | 연도, 동아리명, 전년도 이월금 |
| 월별 수입/지출 | 12개월 데이터 입력 |
| 지출 상세 | 월별 세부 항목 추가 |
| 자동 계산 | 합계, 잔액 실시간 계산 |
| 검증 | 합계 불일치 시 경고 |
| 미리보기 | 손글씨 양식 그대로 렌더링 |
| 이미지 저장 | 캡처해서 카톡 공유 |

### 왜 이렇게 결정했나

| 결정 | 이유 |
|------|------|
| 웹앱 (네이티브 X) | 앱스토어 배포 귀찮음. 링크만 공유하면 됨 |
| PWA 불필요 | 오프라인에서 안 씀 |
| localStorage | 서버 만들면 과함. 연도별로 키 분리 |
| 이미지 저장 | PDF보다 카톡 공유에 편함 |

---

## 🎯 핵심 포인트

### 엄마도 쓸 수 있어야 한다

복잡하면 안 됩니다. 이 기준 하나로 많은 기능이 걸러졌어요.

- 복잡한 설정 → 제거
- 다양한 출력 옵션 → 이미지 하나로 통일
- 로그인/회원가입 → 불필요 (localStorage)

### 반복 작업 = 자동화 기회

"이거 매년 하네?" 하는 순간이 앱 개발의 시작점이었습니다.

**귀찮음을 참지 마세요.**
